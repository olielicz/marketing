🌐 General & Licensing

Applies to all tools
My serial code says "invalid" or won't activate ▾
  1. Double-check you copied the whole code, including all dashes — the format is OLI-XXX-YYYY-ZZZZ-C. A common issue is a trailing space accidentally copied along with it.
  2. Confirm you're entering it into the matching product — a serial code starting OLI-OPS- only works for OliOps Suite, OLI-COM- only for OliCommerce, and so on. Codes aren't interchangeable between products.
  3. Check your internet connection — activation requires one successful check-in with the license server (after that, it works offline for up to 14 days between check-ins).
  4. Still stuck? Contact us with your order confirmation email and we'll verify the code on our end.
"Device limit reached" — I haven't used 5 devices

This usually means an old device is still counted as active — for example, if you reinstalled on the same machine, it may register as a "new" device since the local device ID resets on a clean install.

  1. If you still have access to an old/unused device, open the product there and use the "Deactivate this device" option before decommissioning it.
  2. If you no longer have access to that device (lost laptop, old server already wiped), contact us with your serial code and we'll manually free up a slot — this is a normal, expected request and doesn't require justification.
I didn't receive my serial code after payment
  1. Check your spam/junk folder — automated delivery emails sometimes land there on first send.
  2. Confirm the payment actually completed — check your bank/card statement or PayPal activity for the charge.
  3. Search your inbox for "Oli" or the specific product name.
  4. Still nothing after 30 minutes? Contact billing with your payment confirmation/receipt and we'll resend it immediately.
I want a refund

Lifetime purchases (OliOps, OliCommerce, OliFlow, OliConnect) are covered by a 30-day, no-questions-asked money-back guarantee. Oli-Locator subscriptions can be cancelled anytime with no further charges. See the full Refund Policy for specifics, then email billing with your order details.

Is a specific tool down right now?

Self-hosted tools (OliOps, OliCommerce, OliFlow, OliConnect) run on your own infrastructure, so "down" almost always means your own server/hosting needs a look — check your hosting provider's status page first. Oli-Locator is hosted by us; if you suspect an outage there, contact support and we'll confirm and update you.

💼 OliOps Suite

CRM + Invoicing + AI Support
The server won't start / crashes on launch ▾
  1. Check the terminal output for the actual error message — most startup failures print a clear reason (missing environment variable, port already in use, etc.).
  2. Confirm you're running a supported Node.js version (18+). Check with node --version.
  3. If the error mentions a port conflict, another process may already be using the same port — either stop it or change OliOps's PORT environment variable.
  4. If you just pulled an update, make sure you re-ran npm install (if applicable) before restarting.
I can't log in / forgot my password

Use the "Forgot password" link on the sign-in screen. This works for self-hosted instances too, as long as your instance has outbound email configured (check your setup guide). If you're the very first user on a fresh install and never set a password, check your server console log from the first boot — the initial admin credentials are printed there once.

Invoices/payroll numbers look wrong

Double-check your tax rate settings under Settings → Tax — a new invoice inherits whatever the default tax rate is set to at creation time, so past invoices won't retroactively update if you change the rate later. For payroll, confirm logged hours are entered correctly per pay period; the payroll report calculates strictly from logged-hours × rate.

The AI support chatbot gives a generic or wrong answer

The built-in chatbot resolves common questions from a knowledge base and escalates anything it's not confident about to a real support ticket automatically — so if it can't answer, it should still get a human answer to your customer via the ticket queue. If it's confidently giving wrong information, let us know which question triggered it so we can review the knowledge base.

🛒 OliCommerce Stack

Shopify Automation + AI Assistant
Abandoned cart emails aren't sending ▾
  1. Confirm your Shopify webhook is correctly registered — check your setup guide's webhook configuration step. A missing or misconfigured webhook is the most common cause.
  2. Check your outbound email configuration (SMTP credentials or email provider API key) — cart recovery emails fail silently if the sending credentials are wrong or expired.
  3. Check your server logs for webhook delivery errors around the time a cart should have triggered a recovery email.
The AI shopping assistant isn't showing on my storefront

Confirm the embed script/widget snippet is correctly placed in your theme (usually in the theme's footer or a dedicated app-embed block, depending on your Shopify theme). If you recently changed themes, the embed may need to be re-added since Shopify doesn't always carry app embeds across theme switches automatically.

Supplier CSV forwarding isn't working

Check the destination email/endpoint configured for CSV forwarding — a typo in the supplier's email address is the most common cause. Also confirm new orders are actually triggering the order-paid webhook (see the abandoned-cart troubleshooting above; it's the same underlying webhook dependency).

⚙️ OliFlow Automation Engine

Workflow Automation
A workflow isn't triggering ▾
  1. Confirm the workflow is actually turned on/published — a saved-but-inactive workflow won't trigger. Check the workflow's status toggle.
  2. If it's webhook-triggered: confirm the sending service is hitting the correct webhook URL (check for typos, and confirm you copied the URL for the right workflow — each one has a unique endpoint).
  3. Check the workflow's run history/logs for failed attempts — a failed run usually shows the specific error (e.g. a downstream API returned an error, or a required field was missing from the incoming payload).
A prebuilt template isn't importing correctly

Confirm you're importing the correct file for your OliFlow version — templates are versioned to match specific engine releases. If you recently updated OliFlow, re-download the matching template pack version. If it still fails, send us the exact error message and which template.

REST API calls return 401/403

Confirm your API key is included in the Authorization header and hasn't been regenerated/revoked since you last saved it. API keys can be viewed and regenerated from Settings → API Access — regenerating invalidates the old key immediately, so update any scripts using the old one.

📱 OliConnect

Browser Extension
The extension isn't detecting my top posts ▾
  1. Confirm you're logged into the social platform in the same browser where the extension is installed — OliConnect reads through your existing session, so it needs an active login in that browser.
  2. Give it a moment after connecting a new account — initial post-history detection can take a short scan before results populate.
  3. Try refreshing the platform's page after connecting, then reopening the extension popup.
The extension isn't installed / won't load

Confirm your browser is up to date (OliConnect targets current Chrome/Edge/Firefox versions). If you installed it manually (not via a store listing), confirm Developer Mode / "load unpacked extensions" is enabled in your browser's extension settings.

Is it safe that this reads my social accounts?

Yes — see the full explanation on our Security page. Short version: your login credentials never leave your browser session and are never sent to our servers.

🏡 Oli-Locator

Real Estate CRM (Hosted)
I can't log in ▾

Use "Forgot password" on the login screen — a reset link is sent to your account email. If you want to try the product without your own account first, use the free demo login shown on the Oli-Locator page.

Tapping a lead's number doesn't open my calling app

Oli-Locator opens your device's default calling app via a standard tap-to-call link — if nothing happens, check which app is set as your device's default dialer/calling app in your phone or browser settings. See the "Use any calling app you want" section on the Oli-Locator page for how this works and how to set a default.

Property search results look outdated or missing

Property data refreshes on a regular schedule from the connected listing source — if a specific listing looks stale, it may not have synced yet. If an entire region/city consistently shows no results, let us know which area so we can check the data source coverage there.

I want to cancel my subscription

Cancel anytime from Account Settings → Subscription — no phone call or retention flow required. You'll retain access through the end of your current billing period. See the Refund Policy for billing specifics.
