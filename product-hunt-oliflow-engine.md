I'm olielicz. I built OliFlow because I was tired of watching automation costs scale with usage instead of value — every Zapier/Make task run adds up, and none of it is yours to keep if you ever stop paying.

- 🎨 **Visual workflow builder** — drag-and-drop automation, no code required
- 🔌 **Webhook triggers** — connect any CRM/SaaS that can send a webhook
- 🧩 **Headless REST API** — or skip the UI entirely and run it programmatically
- 📄 **Prebuilt template pack** (via the companion auto-tools repo) — ready-to-import n8n workflows, Make blueprints, Zapier-style configs, and 7 CRM adapters

**Why it's different:**
- **One-time payment, $99.** Not $20–70/month with metered task limits.
- **Self-hosted.** Your workflow data and credentials never leave your own infrastructure.
- **Zero dependencies.** Runs standalone — no external services required to operate the core engine.

**Who it's for:** agencies and ops teams running enough automations that per-task pricing has become a real line item, and who'd rather own the engine outright.
